# FlClash 无图形纯终端部署指南 (Ubuntu 22.04)

## 前提条件

```bash
# 安装 Go 1.22+
wget https://go.dev/dl/go1.22.10.linux-amd64.tar.gz
rm -rf /usr/local/go && tar -C /usr/local -xzf go1.22.10.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
go version  # 确认版本
```

## 文件说明

上传 `flcc/` 整个目录到 Ubuntu 22.04 机器上，目录结构如下：

```
flcc/                      ← 将 flcc 上传到 Linux 后，这里就是工作根目录
├── headless/              # Go 源码 - 终端控制程序 (CLI + TUI)
├── FlClashCore            # 预编译的代理核心引擎 (静态链接, 53MB) ← 必须加执行权限
├── assets/data/           # GEOIP/GEOSITE 路由数据库文件
├── core/                  # 核心 Go 源码
└── ...
```

> **无需 Flutter / Clash.Meta 子模块**。FlClashCore 已预编译为静态二进制，不依赖任何外部运行时。

**所有后续命令中的相对路径 `../xxx` 都相对于 `headless/` 目录。**

---

## 第一步：给 FlClashCore 加执行权限

**当前目录：`flcc/`（项目根目录）**

```bash
# 在 flcc/ 目录下执行
chmod +x FlClashCore
ls -l FlClashCore   # 确认有 -rwxr-xr-x
```

> 忘加会报 `fork/exec ../FlClashCore: permission denied`。也可以复制到系统路径：
> ```bash
> sudo cp FlClashCore /usr/local/bin/
> sudo chmod +x /usr/local/bin/FlClashCore
> ```

---

## 第二步：编译 headless 终端程序

**当前目录：`flcc/headless/`**

```bash
cd headless
go build -o flclash-headless .
ls -l flclash-headless   # 确认生成成功
```

---

## 第三步：确认 GEOIP/GEOSITE 数据文件位置

**不需要复制任何文件**。

`storage/paths.go` 中的 `GetDataDir()` 已修改为自动检测可执行文件同级的 `../assets/data/` 目录。编译后 `flclash-headless` 会自动使用 `headless/../assets/data/`（即 `assets/data/`）作为数据目录，里面的 GEOIP.dat、GEOSITE.dat 等文件直接可用。

```bash
# 确认文件存在即可（不需要复制）
ls ../assets/data/
# 应看到: GEOIP.dat  GEOSITE.dat  GEOIP.metadb  ASN.mmdb
```

---

## 第四步：运行

**当前目录：`flcc/headless/`**

### 启动方式

> **注意**：`sudo ./xxx &` 后台运行时，sudo 需要终端交互输密码，会导致进程被挂起。必须先缓存 sudo 凭证。

```bash
# 先缓存 sudo 密码
sudo -v

# 启动后台 daemon
sudo ./flclash-headless daemon &

# 等1秒后进入 TUI 交互界面
sleep 1
sudo ./flclash-headless tui
```

或者开两个终端（推荐新手）：

```bash
# 终端1：前台运行 daemon（会卡住，不要关）
sudo ./flclash-headless daemon

# 终端2：连接 TUI
sudo ./flclash-headless tui
```

**原理**：`flclash-headless` 分两个角色：

| 角色 | 命令 | 说明 |
|------|------|------|
| **服务端** | `daemon` | 后台运行，创建 Unix Socket，管理 FlClashCore 进程 |
| **客户端** | `tui` / `status` / `start` / `stop` / ... | 通过 Socket 连接 daemon 收发命令 |

必须先启动 daemon，TUI 和 CLI 才能工作。

### TUI 首次使用流程

```text
仪表盘 (按 c) → 配置管理 (按 a) → 输入订阅 URL → 确认导入
返回仪表盘 (按 b) → 启动核心 (按 r) → 确认启动
```

TUI 界面导航：

| 按键 | 功能 |
|------|------|
| `1` | 仪表盘（流量、速度、内存） |
| `2` | 查看并切换代理节点组 |
| `3` | 选择全局出口 |
| `4` | 切换运行模式（规则/全局/直连） |
| `5` | 查看关键日志 |
| `6` / `c` | 管理配置文件和订阅 |
| `r` | 启动/停止核心 |
| `n` | 切换 TUN |
| `q` | 退出 TUI |

### CLI 命令（daemon 运行时）

**当前目录：`flcc/headless/`**

```bash
sudo ./flclash-headless status            # 查看运行状态
sudo ./flclash-headless start             # 启动代理核心
sudo ./flclash-headless stop              # 停止
sudo ./flclash-headless restart           # 重启
sudo ./flclash-headless global <出口名>   # 切换全局出口
sudo ./flclash-headless tun on|off        # TUN 虚拟网卡
sudo ./flclash-headless logs              # 查看日志
```

## 核心启动失败排查

| 现象 | 原因 | 解决 |
|------|------|------|
| `fork/exec ../FlClashCore: permission denied` | FlClashCore 没加执行权限 | 回到 `flcc/` 执行 `chmod +x FlClashCore` |
| `RefreshGroups error: timeout` | 缺少 GEOIP/GEOSITE 数据文件 | 确认 `../assets/data/` 下有这些文件，然后重新编译 `go build -o flclash-headless .` |
| `核心启动失败: ...` | 订阅配置无效或网络不通 | 在 TUI 按 `c` → `a` 重新导入，检查 URL |

## 第四步：配置 systemd 开机自启 (可选)

**当前目录：`flcc/headless/`**

```bash
# 复制二进制到系统目录
sudo cp flclash-headless /usr/local/bin/
sudo cp ../FlClashCore /usr/local/bin/
sudo chmod +x /usr/local/bin/FlClashCore

# 创建 systemd 服务
sudo tee /etc/systemd/system/flclash-headless.service << 'EOF'
[Unit]
Description=FlClash Headless Daemon
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/flclash-headless daemon
Restart=on-failure
RestartSec=3
User=root
WorkingDirectory=/root

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now flclash-headless
journalctl -u flclash-headless -f    # 查看启动日志
```

## 完整架构

```
                        ┌───────────────────────────┐
  flcc/headless/  ──→   │  flclash-headless         │
    ./flclash-headless   │  ├── daemon  (服务端)     │
                         │  ├── tui     (客户端)     │
                         │  ├── status  (客户端)     │
                         │  └── start/stop/...(客户端)│
                         │         │                 │
                         │         │ Unix Socket     │
                         │         ▼                 │
                         │  ┌──────────────────┐    │
                         │  │  FlClashCore     │    │
                         │  │  (Clash.Meta 引擎)│    │
                         │  │  端口 :7890      │    │
                         │  │  控制 :9090      │    │
                         │  └──────────────────┘    │
                         └───────────────────────────┘
```

## 注意事项

1. **执行权限**：FlClashCore 必须有 `chmod +x`
2. **工作目录**：所有命令在 `flcc/headless/` 下执行，相对路径 `../` 指 `flcc/`
3. **TUN 需要 root**：TUN 模式需要 root 权限
4. 默认混合端口 `7890`，外部控制 `127.0.0.1:9090`
5. 更换订阅后需重启核心使配置生效
6. 导入订阅可通过 TUI 配置页（按 `c` → `a`）在线添加 URL
